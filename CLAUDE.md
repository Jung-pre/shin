# 신세계안과 랜딩 — 3D·인터랙션 프로젝트 가이드

## 목표

- **피그마 디자인**을 기준으로 단일 스크롤 랜딩을 구현하고, **3D 모션·스크롤/호버 인터랙션**이 자연스럽게 이어지도록 만든다.
- “보이는 것”뿐 아니라 **프레임 타이밍·관성·스크롤 연동**까지 매끄러운 체감 품질을 우선한다.

## 페이지 구조 (섹션 단위 플랜)

아래 순서는 스크롤 흐름과 구현 우선순위를 함께 고려한 것이다.


| 순서  | 섹션                     | 핵심 과제                                                                                      |
| --- | ---------------------- | ------------------------------------------------------------------------------------------ |
| 1   | **헤더 + 히어로**           | **전역**: `main/common` + `main-page`. **히어로**: `main/sections/hero/main-hero.tsx`. 타 페이지는 `site-page-layout` |
| 2   | **의료진**                | 화이트 베이스 전환, 그룹 이미지·카피 정렬, 스크롤 진입 시 페이드/슬라이트                                                |
| 3   | **시술·기술 그리드**          | 카드 그리드, 호버/포커스 상태, 모션은 과하지 않게                                                              |
| 4   | **첨단 장비 (다크)**         | 블랙 배경·글로우 라인, **장비 3D 또는 고품질 정적+라이트 모션**. 스크롤 연동 시 ScrollTrigger·카메라/오브젝트 이동 설계            |
| 5   | **클리닉 경험·소셜 프루프**      | 이미지 그리드, 워터마크 타이포, lazy load                                                               |
| 6   | **프로세스 패널**            | 히어로와 유사한 **가로 인터랙션** 재사용 가능한 컴포넌트로 추상화                                                     |
| 7   | **블로그·뉴스·이벤트·프로모 CTA** | 카드 레이아웃, SMILE Pro 등 강조 배너                                                                 |
| 8   | **푸터**                 | 상단과 호응하는 그라데이션 마감                                                                          |


## 소스 구조

- **`src/app/`** — 라우터에만 연결된 `page.tsx` / `layout.tsx`. 본문은 피처·컴포넌트를 import만 한다.
- **`src/components/`** — 여러 페이지에서 쓰는 UI (`gnb`, `site-page-layout`, `locale-switcher` 등). 파일 단위로 둔다.
- **`src/features/<영역>/`** — **섹션 단위** 화면·도메인. 해당 영역에만 쓰는 API 클라이언트·훅·맵퍼가 생기면 **같은 폴더에** 둔다 (`lib/api` 분리는 필요할 때만).
- **`src/features/main/main-page.tsx`** — 홈 `<main>` 전체 (`MainPage`: backdrop·글래스·GNB·섹션들).
- **`src/features/main/common/`** — 메인 전역 배경·글래스 (`grid-background`, `glass-orbs-scene`, `use-dom-transmission-texture`).
- **`src/features/main/sections/`** — 스크롤 랜딩 **섹션 단위** 컴포넌트. 예: `sections/hero/`(퀵바·`EyeScene`), `sections/typography/`(타이포 스크롤 섹션).
- **`src/shared/`** — i18n, 전역 설정 등 (`config/`, `i18n/`, `lib/`). `shared/ui`는 사용하지 않는다.

## 기술 스택 (이 레포 기준)

- **Next.js 16** App Router, React 19 — 서버/클라이언트 경계 명확히: Canvas·GSAP·R3F는 `dynamic(..., { ssr: false })` 등 기존 패턴 유지.
- **GSAP + @gsap/react** — 타임라인, `ScrollTrigger`로 스크롤 구간 핀·패럴랙스·가로 스크롤 매핑.
- **Three.js + React Three Fiber + drei** — 장비/히어로 3D, 환경·라이팅, 성능 예산 관리.
- **Framer Motion** — UI 마이크로 인터랙션·레이아웃 전환 보조 (3D와 중복 이펙트는 피한다).

## 완료 기준 (체크리스트)

- 피그마 주요 섹션이 스크롤 순서대로 구현되고 시각적으로 대응된다.
- 히어로·프로세스·장비 구간의 가로/3D 인터랙션이 끊김 없이 동작한다.
- 모바일·태블릿에서 레이아웃 붕괴 없이 사용 가능하다(3D는 폴백 허용).
- `prefers-reduced-motion` 및 키보드 내비게이션이 허용 수준 이상이다.
- `next build` 및 린트 통과.
